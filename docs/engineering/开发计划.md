## P1 — 契约格式基础

**目标**：定义机器可读的契约格式，提供校验能力。

### 交付物

| 文件 | 说明 |
|---|---|
| `schemas/contract-v1.json` | 契约 JSON Schema |
| `examples/auth-pack.json` | auth-pack 样例（User + Role） |
| `packages/contract-core/` | TypeScript 校验库 |

### contract-core API

```typescript
// 从文件加载并校验契约包
loadContractPack(path: string): Promise<ContractPack>

// 从 JSON 字符串校验契约包
validateContractJson(json: string): Result<ContractPack, ContractError>

// 契约包类型
interface ContractPack {
  pack: string
  version: string
  conventions?: Conventions
  entities: Record<string, Entity>
}
```

### 验收标准

- [ ] `examples/auth-pack.json` 通过 Schema 校验
- [ ] 缺失 `pack` 字段的 JSON 被拒绝，错误信息可读
- [ ] 非法字段类型（如 `varchar`）被拒绝
- [ ] 全部单元测试通过

### 技术栈

- TypeScript + Node.js
- `ajv`（JSON Schema 校验）
- `vitest`（测试）

## P2 — MCP Server

**目标**：让 Cursor / Codex / Claude 等 AI 工具能感知本地契约。

### 交付物

| 文件 | 说明 |
|---|---|
| `packages/mcp-server/` | MCP Server 实现 |
| `packages/mcp-server/README.md` | 接入说明（Cursor 如何配置） |

### MCP Tools 设计

```
list_packs        → 列出所有已加载的契约包
get_pack          → 获取某个契约包的完整定义
get_entity        → 获取某个实体的字段和关联
search_entity     → 按名称搜索实体
list_field_types  → 列出某实体所有字段类型
```

### 验收标准

- [ ] `npx synkord-mcp --dir ./contracts` 能启动
- [ ] Cursor MCP 配置接入后，AI 能正确回答"User 实体有哪些字段"
- [ ] 契约文件修改后，MCP Server 热更新（无需重启）

### 接入示例（Cursor .cursor/mcp.json）

```json
{
  "mcpServers": {
    "synkord": {
      "command": "npx",
      "args": ["synkord-mcp", "--dir", "./contracts"]
    }
  }
}
```

## P3 — 桌面 App 骨架

**目标**：Electron App 能浏览本地契约库。

### 交付物

- Electron + React 项目骨架
- 契约包列表页（显示所有加载的 pack）
- 契约详情页（实体列表 + 字段 + 关联）
- 本地契约目录配置

### 验收标准

- [ ] App 启动，能选择本地契约目录
- [ ] 能浏览 auth-pack 的 User 和 Role 实体
- [ ] 页面风格简洁，字段信息清晰可读

---

## P4 — 项目订阅与变更通知

**目标**：注册项目与契约的依赖关系，感知版本变化。

### 交付物

- 项目注册表（SQLite，记录项目 → 契约包订阅关系）
- 版本检测：对比已订阅版本与当前版本
- 通知中心 UI：展示哪些契约有新版本、diff 是什么

### 验收标准

- [ ] 能注册一个项目并订阅 `auth-pack ^1.x`
- [ ] 当 `auth-pack.json` 版本号升级时，App 显示通知
- [ ] 通知展示字段级别的 diff

---

## P5 — 悬浮看板

**目标**：开发者工作时随时查看当前相关契约。

### 交付物

- `alwaysOnTop` 透明悬浮窗
- 显示当前项目的契约摘要
- 支持最小化 / 展开 / 固定到屏幕边缘

### 验收标准

- [ ] 悬浮窗不遮挡 IDE 主内容区
- [ ] 能快速搜索字段名
- [ ] 有新版本时角标提示

---

## P6 — CLI 校验

**目标**：`synkord check` 命令，集成到 CI/CD。

### 交付物

- `packages/cli/` — CLI 工具
- `synkord check <contract-dir>` — 校验契约文件合法性
- GitHub Actions 示例配置

### 验收标准

- [ ] `synkord check examples/` 对有效契约返回 exit 0
- [ ] 对无效契约返回 exit 1，输出可读错误信息
- [ ] GitHub Actions 示例能直接复用
