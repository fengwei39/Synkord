# Synkord 开发计划

> 更新：2026-06  
> 开发方式：全程使用 Cursor，每个任务单复制给 Cursor 直接执行  
> 原则：任务单自包含、粒度小、有明确验收命令

---

## 使用方式

1. 按顺序执行任务单
2. 将任务单全文复制到 Cursor 对话框
3. Cursor 完成后，运行验收命令确认通过
4. 通过后提交，进入下一个任务单

## X1 — 契约格式 Schema + TypeScript 校验库

### 背景
阅读 `docs/product/产品设计.md` 第四章。  
`schemas/contract-v1.json` 和 `examples/auth-pack.json` 已存在。

### 目标
实现 TypeScript 校验库 `packages/contract-core`，验证契约 JSON 是否符合 Schema。

### 交付物
```
packages/contract-core/
├── package.json
├── tsconfig.json
├── src/
│   ├── index.ts          # 导出 loadContractPack, validateContractJson, ContractPack
│   ├── types.ts          # ContractPack, Entity, Field, Relation 类型定义
│   ├── validator.ts      # ajv 校验逻辑
│   └── errors.ts         # ContractError 类
└── tests/
    ├── valid.test.ts
    └── invalid.test.ts
```

### API 要求
```typescript
// 从文件路径加载并校验
export async function loadContractPack(path: string): Promise<ContractPack>

// 从 JSON 字符串校验
export function validateContractJson(json: string): ContractPack  // 失败时 throw ContractError

// 错误类
export class ContractError extends Error {
  constructor(public path: string, public details: string[]) { ... }
}
```

### 依赖
```json
{ "ajv": "^8", "ajv-formats": "^3" }
```
测试：`vitest`

### 验收命令
```bash
cd packages/contract-core
pnpm test        # 全部通过
pnpm build       # 无报错
```

### 验收标准
- [ ] `loadContractPack('../../examples/auth-pack.json')` 返回含 User、Role 的对象
- [ ] 缺少 `pack` 字段的 JSON 抛出 `ContractError`，message 包含 "pack"
- [ ] `type: "varchar"` 抛出 `ContractError`
- [ ] 空 `entities` 对象抛出 `ContractError`

### 不做
- CLI 命令
- 后端集成

## B2 — 邮箱注册与登录 + JWT

### 背景
B1 已完成。阅读 `docs/engineering/开发计划.md` B2 部分。

### 目标
实现邮箱注册、登录，颁发 JWT，提供获取当前用户接口。

### 新增文件
```
server/internal/auth/
├── handler.go     # HTTP handler
├── service.go     # 业务逻辑
├── model.go       # User 结构体
└── middleware.go  # JWT 鉴权中间件
server/migrations/002_users.sql
```

### 数据库表
```sql
CREATE TABLE users (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email        VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255),
  display_name VARCHAR(100) NOT NULL,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE git_emails (
  id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id   UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  email     VARCHAR(255) NOT NULL,
  is_primary BOOLEAN DEFAULT FALSE
);
```

### API
```
POST /api/auth/register
  body: { email, password, displayName }
  resp: { token, user: { id, email, displayName } }

POST /api/auth/login
  body: { email, password }
  resp: { token, user }

GET  /api/auth/me          (需要 Authorization: Bearer <token>)
  resp: { id, email, displayName }

POST /api/auth/git-email   (需要登录)
  body: { email }
  resp: { id, email }
```

### JWT 要求
- 有效期 7 天
- payload 含 `userId`
- 中间件验证失败返回 `401 { "error": "unauthorized" }`

### 验收命令
```bash
cd server && go test ./internal/auth/...
# 手动验证：
curl -X POST localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"123456","displayName":"Test"}'
```

### 验收标准
- [ ] 注册成功返回 token
- [ ] 重复邮箱注册返回 409
- [ ] 错误密码登录返回 401
- [ ] `/api/auth/me` 无 token 返回 401，有 token 返回用户信息

## B4 — 组织创建与邀请系统

### 背景
B2 已完成（需要登录态）。

### 目标
实现组织的创建，以及邀请链接的生成和使用。

### 数据库表
```sql
CREATE TABLE organizations (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name       VARCHAR(100) NOT NULL,
  slug       VARCHAR(50) UNIQUE NOT NULL,
  created_by UUID NOT NULL REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE org_members (
  org_id    UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  user_id   UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role      VARCHAR(20) NOT NULL DEFAULT 'member',  -- 'admin' | 'member'
  joined_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (org_id, user_id)
);

CREATE TABLE org_invites (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id     UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  token      VARCHAR(64) UNIQUE NOT NULL,
  created_by UUID NOT NULL REFERENCES users(id),
  expires_at TIMESTAMPTZ NOT NULL,
  used_by    UUID REFERENCES users(id),
  used_at    TIMESTAMPTZ
);
```

### API
```
POST /api/orgs
  body: { name, slug }
  resp: { id, name, slug }
  → 创建者自动成为 admin

GET /api/orgs/:orgId
  resp: { id, name, slug, memberCount }

GET /api/orgs/me
  resp: [{ id, name, slug, role }]   当前用户所在的所有组织

POST /api/orgs/:orgId/invites        (需要 admin)
  body: { expiresInHours: 72 }
  resp: { token, inviteUrl, expiresAt }

GET /api/invites/:token
  resp: { orgName, inviterName, expiresAt }

POST /api/invites/:token/accept      (需要登录)
  → 加入组织，角色为 member
  resp: { orgId, orgName }
```

### 验收标准
- [ ] 创建组织后 creator 自动成为 admin
- [ ] 邀请 token 唯一，过期后不可用
- [ ] 同一 token 只能使用一次
- [ ] 已是成员的用户接受邀请返回 200（幂等）

## B7 — 契约包 CRUD + 版本 Tag

### 背景
B5、B6、X1（Schema 规则）已完成。

### 目标
实现契约包的增删改查，发布版本时打 Git Tag，内容须通过 JSON Schema 校验。

### API
```
GET  /api/orgs/:orgId/packs
  resp: [{ name, version, updatedAt, ownerEmail }]

POST /api/orgs/:orgId/packs
  body: { name, content }   content 为契约 JSON 字符串
  → 校验 content 符合 contract-v1 JSON Schema
  → 写入 git: {name}/contract.json
  → 打 Tag: {name}/v{version}
  resp: { name, version }

GET  /api/orgs/:orgId/packs/:pack
  resp: { name, version, content }

PUT  /api/orgs/:orgId/packs/:pack
  body: { content }
  → 校验内容，version 必须 > 当前版本
  → 覆盖写入，打新 Tag
  resp: { name, version }

GET  /api/orgs/:orgId/packs/:pack/versions
  resp: [{ version, tagName, committedAt, authorEmail }]

GET  /api/orgs/:orgId/packs/:pack/versions/:version
  resp: { name, version, content }

DELETE /api/orgs/:orgId/packs/:pack    (需要 admin)
  → 删除 git 文件并提交
  resp: 204
```

### Schema 校验
Go 后端嵌入 `schemas/contract-v1.json`（`//go:embed`），用 `qri-io/jsonschema` 或 `xeipuuv/gojsonschema` 校验。

### 验收标准
- [ ] 创建契约包，内容合法 → 201
- [ ] 内容缺少 `pack` 字段 → 400，返回可读错误
- [ ] 发布后 Git Tag 存在：`{pack}/v{version}`
- [ ] 版本号不递增时 PUT 返回 400
- [ ] 全部通过 `go test ./internal/contracts/...`

## B9 — 订阅关系 + WebSocket 通知

### 背景
B7 已完成。

### 目标
用户订阅契约包，发布新版本时通过 WebSocket 推送通知。

### 数据库表
```sql
CREATE TABLE subscriptions (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  org_id           UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  pack_name        VARCHAR(100) NOT NULL,
  project_name     VARCHAR(100),
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, org_id, pack_name, project_name)
);

CREATE TABLE notifications (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  org_id       UUID NOT NULL,
  pack_name    VARCHAR(100) NOT NULL,
  old_version  VARCHAR(50),
  new_version  VARCHAR(50) NOT NULL,
  diff_summary JSONB,
  read_at      TIMESTAMPTZ,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);
```

### API
```
POST /api/orgs/:orgId/packs/:pack/subscribe
  body: { projectName }
  resp: { id }

DELETE /api/orgs/:orgId/packs/:pack/subscribe
  resp: 204

GET /api/notifications?unreadOnly=true
  resp: [{ id, packName, oldVersion, newVersion, diffSummary, createdAt }]

PUT /api/notifications/:id/read
  resp: 204

WS /ws    (Authorization: Bearer <token> as query param)
  → 保持连接，收到 { type: "contract_updated", packName, newVersion, diffSummary }
```

### 发布时触发逻辑
B7 的 `PUT /api/orgs/:orgId/packs/:pack` 成功后，查找所有订阅者，写入 `notifications` 表，并通过 WebSocket 推送。

### 验收标准
- [ ] 订阅后，触发版本发布，通知写入数据库
- [ ] WebSocket 连接中的用户实时收到消息
- [ ] 离线用户重连后能查到未读通知
- [ ] 全部通过 `go test ./internal/notify/...`

## F8 — MCP Server 内嵌

### 背景
F5、B7 已完成。

### 目标
App 启动时自动运行 MCP Server（localhost:3742），AI 工具接入后能查询契约。

### MCP Tools
```
list_packs          列出当前用户订阅的契约包
get_pack            获取某契约包完整定义（JSON）
get_entity          获取某实体字段和关联
search_entity       按名称模糊搜索实体
```

### 实现要点
- 使用 `@modelcontextprotocol/sdk` 创建 MCP Server
- 在 Electron 主进程中以子进程方式启动，或直接集成
- 数据来源：App 本地缓存（SQLite）

### Cursor 接入配置
```json
{
  "mcpServers": {
    "synkord": { "url": "http://localhost:3742/mcp" }
  }
}
```

### 验收标准
- [ ] App 启动后 MCP 端点可访问
- [ ] Cursor 接入后，AI 能正确回答"User 实体有哪些字段"
- [ ] 切换项目后 MCP 返回内容自动更新
