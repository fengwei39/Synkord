# Synkord 产品设计

> 版本：v3.0  
> 更新：2026-06

---

## 一、产品定位

**Synkord 是以业务域为单位的契约管理工具。**

让每个人、每个项目、每个 AI 工具，在任何时候都基于同一份业务数据契约工作。

---

## 二、解决的核心问题

在多人、多项目的团队中，业务数据定义会悄悄漂移：

- 没有一个所有人都认可的、机器可读的数据契约作为基准
- AI 工具（Cursor、Codex 等）生成代码时不知道契约，字段名和类型随意
- 某人修改了共享数据结构，依赖方不知道，联调时才发现问题
- 多个项目对同一个实体的理解各自不同（`userId` vs `user_id` vs `uid`）

---

## 三、核心用户

**核心开发者**：负责多个项目，属于某个团队，项目之间共享业务数据契约。

- 不区分前端 / 后端 / 全栈，只区分**负责的业务域**
- 每个人可以**拥有**某些契约包（负责维护和发布）
- 每个人可以**订阅**其他契约包（依赖消费，感知变更）

---

## 四、核心概念

### 契约包（Contract Pack）

业务域的数据边界声明，机器可读的 JSON 格式。

```json
{
  "pack": "auth-pack",
  "version": "1.2.0",
  "entities": {
    "User": {
      "table": "users",
      "fields": {
        "id":     { "type": "uuid", "primary": true },
        "email":  { "type": "string", "maxLength": 255 },
        "status": { "type": "enum", "values": ["active", "suspended"] }
      },
      "relations": {
        "roles": { "type": "many-to-many", "target": "Role", "through": "user_roles" }
      }
    }
  }
}
```

### 所有权与订阅

| 关系 | 含义 |
|---|---|
| 拥有（Owner） | 负责维护该契约包，有权发布新版本 |
| 订阅（Consumer） | 项目依赖该契约包，感知版本变化 |

### 业务域

一组内聚的契约包，如：

- `auth-pack`：用户与权限域
- `order-pack`：订单与支付域
- `catalog-pack`：商品与库存域

### 组织（Organization）

团队的顶层单位。组织内共享契约包，成员通过邀请制加入。

### 项目配置（synkord.json）

每个项目根目录放置，声明该项目依赖哪些契约包：

```json
{
  "project": "user-service",
  "org": "my-team",
  "consumes": ["auth-pack@^1.x", "order-pack@^2.x"]
}
```

Synkord App 监听该文件，悬浮看板自动切换显示对应契约。

## 五、核心功能

### 5.1 定义契约（Define）

- 用 JSON 格式声明实体、字段类型、关联关系
- 通过 JSON Schema 校验契约文件的合法性
- 版本管理：每次发布更新 `version` 字段，后端打 Git Tag 存档

### 5.2 分发契约（Distribute）

**给 AI 工具**（MCP Server）

Synkord App 内置本地 MCP Server，Cursor / Codex / Claude 等工具接入后，AI 在生成代码时自动感知契约：

```
AI："User 实体有哪些字段？"
Synkord："id(uuid), email(string), status(enum: active/suspended)..."
```

**给开发者**（悬浮看板）

独立悬浮窗常驻桌面（alwaysOnTop），感知当前项目的 `synkord.json`，自动展示相关契约。

### 5.3 感知变化（Notify）

- 契约包发布新版本时，所有订阅项目收到通知
- 通知展示字段级别的 diff：新增 / 删除 / 类型变更
- 开发者决定是否升级、如何适配

## 六、产品形态

### 桌面 App（主入口）

```
安装 Synkord App
    ↓
首次启动：登录（飞书 OAuth 或邮箱）→ 创建 / 加入组织
    ↓
日常：悬浮看板常驻，MCP Server 后台运行
```

桌面 App 包含：
- 悬浮看板（alwaysOnTop，可贴边，可折叠）
- 契约浏览器（组织内所有契约包、实体详情、版本历史）
- 通知中心（版本变更提醒 + diff）
- 设置（账号、Git 邮箱绑定、MCP 端口）

### MCP Server（内置于 App）

App 运行时自动启动，监听 `localhost:3742`，为 AI 工具提供契约上下文。

Cursor 一次性配置：
```json
{
  "mcpServers": {
    "synkord": { "url": "http://localhost:3742/mcp" }
  }
}
```

## 七、账号与组织

### 登录方式

| 方式 | 说明 |
|---|---|
| 飞书 OAuth | 企业团队推荐，一键登录 |
| 邮箱登录 | 邮箱 + 密码，需邮箱验证 |

### 邮箱与 Git 身份绑定

注册邮箱即为主 Git 身份，契约变更提交时使用该邮箱作为 Git 作者，实现变更溯源。支持绑定多个 Git 邮箱（适合一人多身份场景）。

### 组织与邀请

- 任何人可创建组织（成为 Admin）
- Admin 生成邀请链接（含有效期），分发给团队成员
- 成员安装 App 后，输入邀请码或点击链接加入

### 成员角色

| 角色 | 权限 |
|---|---|
| Admin | 管理成员、管理组织内所有契约包 |
| Member | 管理自己拥有的契约包、订阅其他契约包 |

## 八、技术架构

```
┌──────────────────────────────────────────────────┐
│  桌面 App（Electron + React + TypeScript）        │
│  ├── 悬浮看板（alwaysOnTop）                     │
│  ├── MCP Server（localhost:3742/mcp）             │
│  ├── 文件监听（synkord.json）                    │
│  └── 本地缓存（SQLite）                          │
└──────────────────┬───────────────────────────────┘
                   │ HTTPS / WebSocket
┌──────────────────▼───────────────────────────────┐
│  Go 后端                                         │
│  ├── 账号服务（飞书 OAuth + 邮箱 + JWT）          │
│  ├── 组织服务（创建 / 邀请 / 成员 / 角色）        │
│  ├── 契约服务（Git 仓库读写 + 版本 Tag）          │
│  └── 通知服务（WebSocket 推送版本变更）           │
├──────────────────────────────────────────────────┤
│  PostgreSQL（元数据）                            │
│  用户、组织、成员、订阅关系、通知记录             │
├──────────────────────────────────────────────────┤
│  Git 裸仓库（go-git，自托管）                    │
│  每个组织一个仓库，契约包为子目录                │
│  版本 = Git Tag（auth-pack/v1.2.0）              │
└──────────────────────────────────────────────────┘
```

### 技术选型

| 层 | 技术 |
|---|---|
| 后端语言 | Go |
| Web 框架 | Gin |
| 数据库 | PostgreSQL |
| Git 操作 | go-git |
| 桌面外壳 | Electron + React + TypeScript |
| MCP Server | `@modelcontextprotocol/sdk`（Node.js，内嵌于 App） |
| 契约校验 | TypeScript + `ajv` |
| 本地缓存 | SQLite（`better-sqlite3`） |
| 构建工具 | Vite + electron-builder |
| 测试 | Vitest（前端）/ Go test（后端） |
| 包管理 | pnpm workspaces |

### Git 仓库结构

```
orgs/{org-id}/contracts.git
├── auth-pack/
│   └── contract.json
├── order-pack/
│   └── contract.json
└── catalog-pack/
    └── contract.json
```

版本发布时打 Tag：`auth-pack/v1.0.0`、`auth-pack/v1.2.0`

## 九、MVP 范围

**目标：跑通完整核心闭环**

1. 飞书或邮箱登录，创建 / 加入组织
2. 在组织内创建契约包，发布版本
3. MCP Server 启动，Cursor 接入后 AI 能感知契约
4. 悬浮看板感知 `synkord.json`，展示当前项目契约
5. 契约发布新版本时，订阅成员收到通知

**不在 MVP 范围**

- CLI `synkord check`（CI/CD 集成）
- AI 辅助生成契约
- 代码合规 AST 校验
