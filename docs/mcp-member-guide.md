# Synkord 契约集成员管理指南

> 面向契约集创建者（owner）。讲解如何管理团队成员的访问权限。

---

## 角色说明

每个契约集有三种角色：

| 角色 | 权限 | 谁能设为 |
|---|---|---|
| **owner（创建者）** | 全部权限，包括删除契约集、管理成员 | 仅创建者本人，不可变更 |
| **editor（编辑者）** | 增删改 API/数据模型；不可管理成员 | owner 邀请/提升 |
| **viewer（查看者）** | 只读，可通过 MCP 查询 | owner 邀请/提升 |

**硬约束**：创建者不可被移除或降级（后端硬保护 + 前端 UI 隐藏）。

---

## 邀请成员

1. 进入契约集详情
2. 点击顶部 **成员** Tab → **邀请成员**
3. 在搜索框输入用户名（至少 2 个字符）
4. 从结果中选择用户
5. 选择角色（**viewer** 或 **editor**，owner 不可直接邀请）
6. 点击 **添加**

邀请的用户立即生效，无需确认。

---

## 修改成员角色

1. 进入成员管理页面
2. 在「角色」列直接下拉修改（仅 owner 可操作）
3. 选择新角色后自动保存

下拉菜单只显示可变更的角色（创建者的角色不可改）。

---

## 移除成员

1. 在成员列表中点击 **移除** 按钮
2. 弹窗确认
3. 确认后立即生效，被移除用户失去所有访问权限

---

## 权限边界

| 操作 | owner | editor | viewer |
|---|---|---|---|
| 查看契约集 | ✓ | ✓ | ✓ |
| 编辑契约集信息 | ✓ | ✓ | ✗ |
| 删除契约集 | ✓ | ✗ | ✗ |
| 增/改/删 API | ✓ | ✓ | ✗ |
| 增/改/删 数据模型 | ✓ | ✓ | ✗ |
| 导入 OpenAPI | ✓ | ✓ | ✗ |
| 设置为活跃契约集 | ✓ | ✓ | ✓ |
| 通过 MCP 查询 | ✓ | ✓ | ✓ |
| 查看成员列表 | ✓ | ✓ | ✓ |
| 邀请/修改/移除成员 | ✓ | ✗ | ✗ |

## 最佳实践

### 团队规模 vs 角色分配

- **小型团队（< 5 人）**：所有人 owner 即可
- **中型团队（5-20 人）**：核心 1-2 人 owner，其他 editor
- **大型团队**：按模块分 owner，全员 viewer 或 editor

### 临时访问

对于外部合作方或临时成员，建议给 viewer 角色 + 短时间访问（用完移除）。

### 权限转移

如果 owner 要转交给别人：
1. 先将目标用户提升为 editor（如果还不是）
2. **暂不支持直接转移 owner**（需后端配合）
3. 当前方案：联系管理员在后端直接调整

### 审计

所有成员变更会记录在 Synkord 后端审计日志中（未来可在「访问日志」查看）。

## 常见问题

### Q1: 我是 owner，想把另一个用户也设为 owner

**当前不支持**。为避免误操作导致多 owner，邀请时只有 viewer / editor 选项。

如确有需要，请联系 Synkord 管理员在后端调整。

### Q2: 误删了成员怎么办？

可以重新邀请（只要他们还有 Synkord 账号）。

### Q3: 成员看不到契约集？

检查：
1. 成员是否在「成员」列表里
2. 成员角色是否是 viewer/editor（被禁用账号 = 无权限）
3. 成员的 Synkord 账号是否有效
