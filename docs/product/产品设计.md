# Synkord 产品设计文档

> 版本：v4.0 · 更新于 2026-06-10

---

## 一、产品定位

**Synkord** 是一个跨项目文件版本管理与变更通知工具。

> 核心思想：任何需要在多个项目/团队之间共享且需要追踪变更的文本文件，都可以成为"契约"。

### 典型契约文件类型

| 类型 | 示例 |
|------|------|
| 接口文档 | API README.md、OpenAPI YAML |
| 类型定义 | TypeScript interfaces、Go structs |
| 数据协议 | Protobuf .proto、Avro schema |
| 配置约定 | 环境变量说明、枚举值文档 |
| 数据库约定 | SQL 建表语句、字段说明 |
| 任意文本 | 自由格式说明文档 |

---

## 二、核心功能

```
定义契约  ──→  管理版本  ──→  通知变更  ──→  AI 感知
  ↑
可从多项目多文件导入
```

### 2.1 定义（Define）

- 创建"契约包"（Pack），内容为**任意文本文件**（可由多个项目、多个文件组合而成）
- 支持从本地文件系统导入：单文件 / 跨项目多文件选择
- 目录选择时自动递归读取所有子文件，每个文件以 `# 项目名/相对路径` 为标题
- 版本号使用语义化版本（SemVer）：`major.minor.patch`
- 内容类型（`contentType`）：`markdown` / `yaml` / `json` / `typescript` / `go` / `sql` / `proto` / `text`

### 2.2 分发（Distribute）

- **MCP Server**（嵌入 Electron，监听 `localhost:3742`）：AI 工具（Cursor、Copilot 等）通过 MCP 协议查询契约包原始内容，注入上下文
- **悬浮看板**：始终置顶浮动窗口，实时展示当前项目消费的契约包内容，支持关键字搜索
- **IDE 自动同步**：发布新版本后，自动写入关联本地项目的 IDE 配置文件（Cursor / VSCode / Claude / Codex）

### 2.3 通知（Notify）

- 版本发布后，WebSocket 实时推送给订阅该契约包的组织成员
- 通知中心展示未读/历史通知，支持标记已读

## 三、本地项目集成

### 工作流程

```
1. 在 Synkord 关联本地项目目录（可关联多个项目）
2. 浏览文件树，跨项目勾选目标文件/目录
3. 选择发布到哪个契约包（新建或更新版本）
4. 一键发布为新版本 → 自动通知订阅者 → 自动同步 IDE 文件
```

### 数据存储

- 本地项目映射存储在 `localStorage`（纯本地，不上传）
- 绑定记录（`projectId → packName → filePath`）同样本地存储
- 契约包实际内容存储在后端 Git 裸仓库

### .synkord/config.json（IDE 集成配置）

同步后，在项目根目录自动生成 `.synkord/config.json`：

```json
{
  "orgId": "...",
  "orgSlug": "my-org",
  "project": "frontend-app",
  "consumes": ["user-api", "order-schema"],
  "lastSyncedAt": "2026-06-10T12:00:00Z"
}
```

同时按需生成 IDE 特定文件（仅当对应 IDE 目录存在时才写入）：

| 文件 | 条件 | 作用 |
|------|------|------|
| `.cursor/rules/synkord.md` | `.cursor/` 存在 | Cursor 自动加载的契约规则 |
| `.cursor/mcp.json` | `.cursor/` 存在 | Cursor MCP 服务器配置 |
| `.vscode/mcp.json` | `.vscode/` 存在 | VS Code Copilot MCP 配置 |
| `CLAUDE.md` | 文件已存在 | Claude Code 上下文注入 |
| `AGENTS.md` | 文件已存在 | Codex/OpenAI Agents 上下文 |

> 写入前会比对内容，内容无变化时跳过，避免触发不必要的 IDE 重载。

## 四、技术架构

### 客户端（Electron）

```
┌─────────────────────────────────────┐
│  Electron Desktop App               │
│                                     │
│  ┌─────────────────┐  ┌──────────┐  │
│  │   主窗口         │  │ 悬浮看板  │  │
│  │  React + Router │  │ alwaysOnTop│  │
│  │  契约浏览/编辑   │  │ 内容预览  │  │
│  │  本地项目管理    │  │ 关键字搜索│  │
│  └─────────────────┘  └──────────┘  │
│                                     │
│  ┌─────────────────────────────┐    │
│  │   MCP Server (port 3742)    │    │
│  │   JSON-RPC over HTTP        │    │
│  │   tools: list_packs         │    │
│  │          get_pack           │    │
│  │          search_pack        │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

### 后端（Go + Gin）

```
API Server (port 8080)
├── /api/auth/                           # 登录/注册/Profile/Git 邮箱
├── /api/orgs/                           # 组织管理（创建/邀请/成员）
├── /api/orgs/:id/register-device        # 设备注册（自动订阅所有包）
├── /api/orgs/:id/packs/                 # 契约包 CRUD
├── /api/orgs/:id/packs/:p/versions      # 版本历史
├── /api/orgs/:id/packs/:p/diff          # 行级 diff
├── /api/orgs/:id/packs/:p/subscribers   # 使用者管理
├── /api/notifications/                  # 通知列表/已读
└── /ws                                  # WebSocket 实时通知
```

**契约存储**：每个组织一个 Git 裸仓库，契约包目录下存放 `contract` 文件（任意文本），版本通过 Git Tag（`packName/v1.0.0`）管理。

### 数据库（PostgreSQL）

| 表 | 用途 |
|----|------|
| `users` | 用户账号 |
| `user_git_emails` | 用户 Git 邮箱（可多个） |
| `organizations` | 组织 |
| `org_members` | 组织成员（含角色） |
| `org_invites` | 邀请链接 |
| `contract_packs` | 契约包元数据（含 `content_type`） |
| `subscriptions` | 订阅关系（含设备/Git/项目信息） |
| `notifications` | 通知记录 |

## 五、diff 格式（行级）

`GET /api/orgs/:orgId/packs/:pack/diff?from=1.0.0&to=1.1.0`

```json
{
  "from": "1.0.0",
  "to": "1.1.0",
  "stats": { "added": 3, "removed": 1 },
  "hunks": [
    {
      "oldStart": 10,
      "newStart": 10,
      "lines": [
        { "type": "context", "oldNum": 10, "newNum": 10, "content": "  email: string" },
        { "type": "removed", "oldNum": 11, "content": "  age: number" },
        { "type": "added",   "newNum": 11, "content": "  age: number // 必须 > 0" },
        { "type": "added",   "newNum": 12, "content": "  phone: string" }
      ]
    }
  ]
}
```

## 六、MCP 工具列表

AI 通过 MCP Server 可调用：

| 工具 | 参数 | 说明 |
|------|------|------|
| `list_packs` | `org_id` | 列举组织下所有契约包（名称、版本、类型） |
| `get_pack` | `org_id`, `pack_name` | 获取契约包完整原始内容（任意文本） |
| `search_pack` | `org_id`, `pack_name`, `query` | 在契约包内容中全文搜索，返回匹配行 |

AI 通过注入原始文本内容感知最新契约状态，实现"消除信息不对称"的目标。

## 七、使用者（Subscribers）

加入组织后，客户端启动时自动调用 `POST /api/orgs/:orgId/register-device`，将当前用户注册为所有契约包的使用者，并上报设备信息。

使用者列表展示：

| 字段 | 来源 |
|------|------|
| 邮箱 | users 表 |
| 固定版本 / 是否最新 | subscriptions.pinned_version vs 当前版本 |
| 设备信息 | OS 平台、主机名、用户名 |
| Git 邮箱 | user_git_emails 表 + 设备上报 |
| 本地项目 | 客户端上报的项目名列表 |
| 上次同步时间 | subscriptions.updated_at |

管理员可手动添加（按邮箱）或移除使用者。
