# Synkord 原型结构与页面关系文档

本文档用于后续重构当前项目，约定 Synkord 的页面层级、菜单关系、入口逻辑、核心页面模块和交互边界。后续以前端代码、需求文档和本文档作为主要依据推进。

## 1. 产品信息架构

Synkord 是开源、自托管的 MCP 规范协同平台。当前阶段的产品层级如下：

```text
Synkord
├─ 启动链路
│  ├─ 后端连接配置
│  └─ 登录
├─ 工作台
│  ├─ 我的团队
│  ├─ 最近访问项目
│  └─ 创建团队引导
└─ 我的团队（当前团队上下文）
   ├─ 项目管理
   │  ├─ 项目列表
   │  └─ 项目详情
   │     ├─ 项目信息
   │     ├─ 接口管理
   │     ├─ 数据模型
   │     ├─ 依赖拓扑
   │     └─ MCP
   └─ 团队管理
      ├─ 团队信息
      └─ 成员与权限
```

核心原则：

- 产品只保留“我的团队”，团队是最高业务容器和数据隔离边界。
- 团队层只提供“项目管理”和“团队管理”两个业务入口。
- 接口管理、数据模型、依赖拓扑和 MCP 管理必须进入具体项目后使用。
- 当前阶段不设计变更检测、变更记录、变更通知或 Webhook 通知模块。
- 用户首次登录后如果没有团队，必须先创建团队。
- 后端连接配置属于客户端本地配置，不是登录后的全局业务设置。
- MCP 配置只存在于当前项目上下文内；Electron 同一时间只激活一个团队和一个项目。

## 2. 全局入口逻辑

### 2.1 首次启动

```text
启动 Electron
↓
检查是否配置 synkord-core 后端地址
↓
未配置：进入后端连接配置页
已配置：检查登录态
↓
未登录：进入登录页
已登录：检查是否已有团队
↓
无团队：进入创建团队引导页
有团队：进入上次访问团队的项目列表
```

### 2.2 新用户首次进入

```text
配置后端地址
↓
登录
↓
无团队
↓
创建团队
↓
进入当前团队的项目列表
```

### 2.3 老用户正常进入

```text
启动客户端
↓
检查后端地址
↓
检查登录态
↓
进入上次访问团队
↓
默认打开项目列表或上次访问项目详情
```

### 2.4 团队切换

```text
点击团队
↓
切换 currentTeamId
↓
清空 currentProjectId
↓
刷新项目列表和团队成员
↓
进入当前团队的项目列表
```

切换团队后只刷新：

- 项目列表。
- 团队信息。
- 团队成员与权限。
- Electron 当前激活 MCP 上下文，必要时清空当前激活项目。

### 2.5 团队入口规则

- 登录后如果用户没有任何团队，只显示创建团队引导页。
- 创建团队成功后，创建者自动成为该团队管理员，并进入项目列表。
- 如果用户有多个团队，默认进入上次访问团队；没有访问记录时进入列表中的第一个团队。
- 团队菜单只显示项目管理和团队管理。
- 未进入项目时，不显示接口管理、数据模型、依赖拓扑和 MCP 页面。
- 删除或退出当前团队后，如果仍有其他团队，自动切换到下一个可访问团队；如果没有团队，回到创建团队引导页。

## 3. 全局布局

### 3.1 顶部导航

用途：承载产品入口、当前团队、当前激活项目、本地 MCP 服务状态和用户入口。

组件：

- Synkord Logo / 产品名。
- 当前团队选择器。
- 当前激活项目提示。
- 本地 MCP 服务状态。
- 当前用户头像 / 用户菜单。
- Electron 窗口控制按钮。

字段：

- 当前团队名称。
- 当前激活项目名称；未进入项目时显示未激活。
- 本地 MCP 服务状态：运行中 / 已停止 / 未配置 / 异常。
- 当前登录用户。

交互：

- 点击 Logo 回到工作台。
- 切换团队后进入项目列表，并清空项目内页面上下文。
- 点击本地 MCP 服务状态查看本机服务状态、端点、日志和启动操作。
- 点击用户头像展开个人信息、退出登录。

### 3.2 侧边菜单

菜单结构：

```text
左侧菜单
├─ 我的团队
│  ├─ 默认团队
│  ├─ Team A
│  └─ Team B
├─ 新建团队
└─ 团队功能菜单
   ├─ 项目管理
   └─ 团队管理
```

菜单关系：

| 菜单 | 跳转页面 | 依赖上下文 |
| --- | --- | --- |
| 项目管理 | 项目列表 | 当前团队 |
| 团队管理 | 团队信息、成员与权限 | 当前团队 |

交互：

- 点击团队切换当前团队上下文。
- 点击项目管理进入项目列表。
- 点击团队管理进入团队信息和成员管理。
- 当前菜单高亮。
- 无权限菜单隐藏或置灰。

### 4.6 项目 MCP 管理

用途：作为项目详情中的 MCP 能力页，管理当前项目如何通过本机唯一 MCP 服务暴露给 IDE/Agent。用户打开哪个项目，IDE 连接 Synkord MCP 时就读取哪个项目的规范。

页面结构：

```text
项目详情
└─ MCP
   ├─ MCP 概览
   ├─ 本地服务状态
   ├─ 当前项目接入配置
   ├─ Token 管理
   ├─ 工具列表
   ├─ IDE 接入说明
   └─ 调用审计
```

入口关系：

| 来源 | 操作 | 去向 |
| --- | --- | --- |
| 项目详情 | 打开 MCP Tab | 项目 MCP 管理 |
| 项目 MCP 管理 | 设为当前 MCP 项目 | 更新本地 MCP 激活上下文 |
| 项目 MCP 管理 | 启动本地 MCP 服务 | 本地服务状态变为运行中 |
| 项目 MCP 管理 | 新建 Token | Token 弹窗 |
| 项目 MCP 管理 | 轮换 Token | 旧 Token 立即失效 |
| 项目 MCP 管理 | 查看调用审计 | 审计记录 |
| 项目 MCP 管理 | 复制 IDE 配置 | IDE 接入说明 |

MCP 可用性设计：

- 当前设备只运行一个 Synkord 本地 MCP 服务入口。
- 本地 MCP 服务同一时间只绑定 Electron 当前打开的一个团队和一个项目。
- 打开项目详情的 MCP Tab 时，Electron 自动把该项目设为当前 MCP 激活项目。
- IDE 配置中的本地 MCP 地址保持稳定；项目切换不要求用户修改 `.mcp.json`。
- 每个项目可以创建多个 MCP Token，用于区分 Codex、Cursor、VSCode、PyCharm、JetBrains 等不同 IDE/Agent。
- Token 只能访问创建它的项目，不能跨团队或跨项目使用。

MCP 配置字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| name | 文本 | 配置名称，例如 Cursor 开发环境 |
| purpose | 文本 | 用途说明 |
| token | 密钥 | MCP 访问 Token，仅创建或轮换时完整展示 |
| team_id | 关联字段 | 所属团队 |
| project_id | 关联字段 | 所属项目 |
| tool_scope | 多选 | 允许调用的 MCP 工具 |
| enabled | 状态 | 启用 / 停用 |
| expires_at | 时间 | 过期时间，可为空 |
| last_used_at | 时间 | 最近调用时间 |
| created_by | 用户 | 创建人 |
| created_at | 时间 | 创建时间 |

MCP 访问状态规则：

| 本地 MCP 服务 | 当前项目上下文 | Token 状态 | 访问结果 |
| --- | --- | --- | --- |
| 已停止 | 任意 | 任意 | 拒绝访问 |
| 运行中 | 未激活项目 | 任意 | 拒绝访问 |
| 运行中 | 已激活项目 | 停用或过期 | 拒绝访问 |
| 运行中 | 已激活项目 | 启用且未过期 | 允许访问授权工具 |

IDE 配置示例：

```json
{
  "mcpServers": {
    "synkord": {
      "url": "${SYNKORD_MCP_URL}",
      "headers": {
        "Authorization": "Bearer ${SYNKORD_MCP_TOKEN}"
      }
    }
  }
}
```

该配置只连接本地 MCP 服务，不表达团队或项目。团队和项目由 Electron 当前激活上下文决定。

## 5. 核心业务流

### 5.1 项目到接口 / 模型 / 依赖 / MCP

```text
项目列表
↓
项目详情
├─ 接口管理：查看、创建、导入、导出当前项目 API
├─ 数据模型：查看、创建、编辑当前项目模型
├─ 依赖拓扑：查看当前项目接口、模型和跨项目引用关系
└─ MCP：设置当前项目为本地 MCP 激活项目并管理 Token
```

### 5.2 接口到模型

```text
接口详情
├─ 请求模型 → 模型详情
├─ 响应模型 → 模型详情
├─ 被哪些项目使用 → 项目详情
└─ 查看引用关系 → 依赖拓扑
```

### 5.3 模型到引用关系

```text
模型详情
├─ 查看引用接口 → 接口详情
├─ 查看依赖项目 → 项目详情
└─ 查看拓扑 → 依赖拓扑
```

### 5.4 导入 Swagger / OpenAPI / Postman

```text
项目详情
↓
接口管理
↓
点击导入 Swagger / OpenAPI / Postman
↓
上传文件或填写 URL
↓
解析
↓
导入预览
↓
确认导入
↓
生成当前项目接口、模型、引用关系
↓
进入导入结果页
↓
可查看接口列表、模型列表或依赖拓扑
```

### 5.5 MCP 消费闭环

```text
项目详情 MCP Tab
↓
Electron 将该项目设为当前 MCP 激活项目
↓
启动或复用本机唯一 MCP 服务
↓
创建当前项目 MCP Token
↓
IDE 连接本地 MCP 服务
↓
本地 MCP 服务携带 Token + 当前团队项目上下文调用后端 /api/mcp/*
↓
IDE 获取当前项目接口、模型和依赖上下文
```

## 6. 闭环核查清单

| 环节 | 输入 | 输出 | 是否闭环 |
| --- | --- | --- | --- |
| 启动与登录 | synkord-core 地址、账号密码 | 登录态、当前用户 | 是 |
| 团队上下文 | 当前用户 | 当前团队、团队角色、团队菜单 | 是 |
| 团队项目管理 | 项目元数据 | 团队内项目列表和项目详情 | 是 |
| 团队管理 | 团队信息、成员、角色 | 成员权限生效 | 是 |
| 项目接口管理 | Swagger / OpenAPI / Postman 或手动录入 | 当前项目接口资产 | 是 |
| 项目数据模型 | JSON Schema / OpenAPI Schema | 当前项目模型资产 | 是 |
| 项目依赖拓扑 | 接口、模型、项目引用 | 当前项目依赖关系 | 是 |
| MCP 消费 | 当前项目 Token、工具范围 | IDE/Agent 可查询当前项目规范 | 是 |
| 权限兜底 | 平台角色、团队角色 | 菜单、按钮、接口权限控制 | 是 |

## 8. 后续重构优先级

建议按以下顺序重构：

1. 统一全局布局：顶部导航、团队选择器、侧边菜单。
2. 补齐团队与成员模型、创建团队引导和团队切换逻辑。
3. 收敛团队菜单，只保留项目管理和团队管理。
4. 重构项目列表和项目详情。
5. 将接口管理迁入项目详情。
6. 将数据模型迁入项目详情。
7. 将依赖拓扑迁入项目详情。
8. 将 MCP 管理迁入项目详情，并接入 Electron 本地 MCP 服务状态。
9. 补齐团队成员与权限。
10. 按权限模型统一控制菜单、按钮和接口访问。
