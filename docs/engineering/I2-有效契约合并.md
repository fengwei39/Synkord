## 三、项目配置格式（project-v1）

```json
{
  "$schema": "https://synkord.dev/schema/project-v1.json",
  "project": "ecommerce-api",
  "imports": [
    {
      "pack": "auth-pack",
      "file": "../examples/auth-pack.json",
      "version": "^1.0.0"
    }
  ],
  "extensions": [
    {
      "entity": "User",
      "add_fields": {
        "last_login_ip": { "type": "string", "maxLength": 45 }
      },
      "add_relations": {}
    }
  ],
  "local_entities": {
    "AdminAuditLog": {
      "table": "admin_audit_logs",
      "fields": {
        "id":         { "type": "uuid", "primary": true },
        "action":     { "type": "string", "maxLength": 100 },
        "performed_at": { "type": "datetime" }
      }
    }
  }
}
```

### imports 字段

| 字段 | 说明 |
|---|---|
| `pack` | 契约包名，用于日志与报错 |
| `file` | 契约包 JSON 文件路径（相对项目配置或绝对路径） |
| `version` | semver 范围：`1.0.0`（精确）、`^1.0.0`（兼容）、`~1.0.0` |

### extensions 规则

| 操作 | 允许 |
|---|---|
| `add_fields` | ✅ 给共享实体加字段 |
| `add_relations` | ✅ 给共享实体加关联 |
| 修改已有字段名/类型 | ❌ 报错 `ExtensionViolation` |
| 删除已有字段 | ❌ 报错 `ExtensionViolation` |

## 四、Rust API 设计

```rust
/// 项目配置
pub struct ProjectConfig {
    pub project: String,
    pub imports: Vec<PackImport>,
    pub extensions: Vec<EntityExtension>,
    pub local_entities: HashMap<String, Entity>,
}

/// 有效契约（合并结果）
pub struct ResolvedContract {
    pub project: String,
    pub sources: Vec<PackSource>,       // 引用了哪些包及版本
    pub conventions: Conventions,        // 合并后的约定（冲突时报错）
    pub entities: HashMap<String, Entity>, // 合并后的全部实体
}

/// 加载项目配置
pub fn load_project_config(path: &Path) -> Result<ProjectConfig, ContractError>;

/// 解析有效契约
pub fn resolve(project: &ProjectConfig) -> Result<ResolvedContract, ContractError>;
```

### 新增错误变体

```rust
pub enum ContractError {
    // ... I1 已有变体
    VersionMismatch { pack: String, required: String, actual: String },
    ExtensionViolation { entity: String, message: String },
    EntityConflict { entity: String, pack_a: String, pack_b: String },
    ConventionConflict { field: String, value_a: String, value_b: String },
}
```

## 五、resolve 算法

```
1. 按 imports 顺序加载各契约包
2. 校验每个包的 version 满足 imports.version 范围
3. 合并 conventions（不一致 → ConventionConflict）
4. 合并 entities（不同包有同名实体且字段冲突 → EntityConflict）
5. 应用 extensions（违反规则 → ExtensionViolation）
6. 合并 local_entities（与共享实体同名 → EntityConflict）
7. 输出 ResolvedContract
```

### 版本匹配规则

| 写法 | 语义 |
|---|---|
| `1.2.0` | 精确匹配 |
| `^1.2.0` | >= 1.2.0 且 < 2.0.0 |
| `~1.2.0` | >= 1.2.0 且 < 1.3.0 |
