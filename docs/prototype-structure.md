# Synkord 原型结构与页面关系文档

本文档用于后续重构当前项目，覆盖 Synkord 的页面层级、菜单关系、入口逻辑、核心页面模块、字段与交互规则。当前阶段不再依赖 Penpot、Figma MCP 或 TalkToFigma，后续以前端代码和本文档作为主要依据推进。

## 1. 产品信息架构

Synkord 是开源、自托管的 MCP 规范协同平台。产品层级如下：

```text
Synkord
├─ 启动链路
│  ├─ 后端连接配置
│  └─ 登录
├─ 工作台
│  ├─ 我的团队概览
│  ├─ 最近访问项目
│  ├─ 最近变更
│  └─ 创建团队引导
├─ 我的团队（当前团队上下文）
│  ├─ 团队首页
│  ├─ 项目管理
│  │  ├─ 项目列表
│  │  └─ 项目详情
│  ├─ 接口管理
│  │  ├─ 接口列表
│  │  ├─ 接口详情
│  │  ├─ Swagger / Postman 导入
│  │  └─ 导入结果
│  ├─ 数据模型
│  │  ├─ 模型列表
│  │  └─ 模型详情
│  ├─ 项目 MCP 管理
│  │  ├─ MCP 概览
│  │  ├─ Token 管理
│  │  ├─ IDE 接入说明
│  │  └─ 调用审计
│  ├─ 依赖拓扑
│  │  ├─ 拓扑图
│  │  └─ 影响范围分析
│  ├─ 变更检测
│  │  └─ 检测结果
│  ├─ 变更记录
│  │  ├─ 变更列表
│  │  └─ 变更详情
│  ├─ 顶部通知入口
│  │  └─ 通知抽屉
│  └─ 团队成员与权限
│     ├─ 成员列表
│     ├─ 邀请 / 新增成员
│     └─ 编辑 / 禁用 / 删除确认
```

核心原则：

- 产品只保留“我的团队”，团队是最高业务容器。
- 团队是业务数据隔离边界。
- 用户首次登录后如果没有团队，必须先创建团队。
- 用户可以创建多个团队，也可以被邀请加入其他团队。
- 每个团队拥有独立的项目、接口、数据模型、依赖关系、变更记录和成员权限；MCP Token 归属当前项目。
- 依赖拓扑是团队下的一级模块，用于查看当前团队内项目、接口、数据模型之间的依赖和影响范围。
- 登录后必须存在当前团队上下文才能进入团队业务页面。
- MCP 配置只存在于当前项目上下文内。
- 后端连接配置属于客户端本地启动/登录前配置，不作为登录后的全局业务配置。

## 2. 全局入口逻辑

### 2.1 首次启动

```text
启动 Electron
↓
检查是否配置 synkord-core 后端地址
↓
未配置：进入后端连接配置页
已配置：检查登录态
↓
未登录：进入登录页
已登录：检查是否已有团队
↓
无团队：进入创建团队引导页
有团队：进入上次访问团队或默认团队首页
```

### 2.2 新用户首次进入

```text
配置后端地址
↓
登录
↓
无团队
↓
创建团队
↓
进入团队首页
```

### 2.3 老用户正常进入

```text
启动客户端
↓
检查后端地址
↓
检查登录态
↓
进入上次访问团队
↓
默认打开团队首页或上次访问页面
```

### 2.4 团队切换

```text
点击左侧团队
↓
切换 currentTeamId
↓
刷新当前团队上下文
↓
进入该团队首页
↓
所有团队级数据重新加载
```

切换团队后需要刷新：

- 项目列表
- 接口列表
- 数据模型
- MCP Token
- 依赖拓扑
- 变更记录
- 团队成员

### 2.5 团队入口规则

- 登录后如果用户没有任何团队，只显示创建团队引导页，不显示项目、接口、数据模型等团队业务菜单。
- 创建团队成功后，自动将创建者设为该团队管理员，并进入团队首页。
- 如果用户有多个团队，默认进入上次访问团队；没有访问记录时进入列表中的第一个团队。
- 切换团队只影响团队级业务数据和本地 MCP 服务的激活上下文，不影响登录态和本地 MCP 服务进程本身。
- 删除或退出当前团队后，如果仍有其他团队，自动切换到下一个可访问团队；如果没有团队，回到创建团队引导页。

### 2.6 树结构链路核查

每条链路按“父节点 → 入口条件 → 处理节点 → 终点 → 回流位置”检查，要求没有孤立页面、没有无出口操作、没有跨团队数据泄漏。

| 父节点 | 入口条件 | 处理节点 | 终点 | 回流位置 | 结论 |
| --- | --- | --- | --- | --- | --- |
| 启动链路 | 打开 Electron | 后端连接配置 | 地址校验成功保存；失败停留并提示 | 登录页或当前页 | 有始有终 |
| 启动链路 | 未登录或登录态失效 | 登录 | 登录成功获取当前用户；失败停留并提示 | 团队检查或登录页 | 有始有终 |
| 工作台 | 登录后无团队 | 创建团队引导 | 创建成功并成为团队管理员；失败停留并提示 | 团队首页或当前页 | 有始有终 |
| 工作台 | 登录后已有团队 | 我的团队概览 / 最近访问 | 选择当前团队 | 团队首页或上次访问页 | 有始有终 |
| 我的团队 | 点击团队列表 | 团队切换 | currentTeamId 更新、团队数据刷新 | 团队首页 | 有始有终 |
| 我的团队 / 项目管理 | 新建、编辑、归档项目 | 项目列表 / 项目详情 | 项目资产更新 | 项目列表或项目详情 | 有始有终 |
| 我的团队 / 接口管理 | 上传 Swagger / Postman | 导入、解析、预览、确认 | 生成接口、模型、引用关系 | 导入结果 | 有始有终 |
| 我的团队 / 数据模型 | 新建或编辑模型 | 模型列表 / 模型详情 | 模型版本和引用关系更新 | 模型详情 | 有始有终 |
| 我的团队 / 依赖拓扑 | 从项目、接口、模型、变更结果进入 | 拓扑筛选、节点查看、影响分析 | 输出影响范围和风险列表 | 来源页面或拓扑页 | 有始有终 |
| 我的团队 / 变更检测 | 从导入结果或项目详情进入 | 对比新旧 OpenAPI / JSON Schema | 生成检测结果 | 变更结果页 | 有始有终 |
| 我的团队 / 变更记录 | 保存检测结果 | 变更列表 / 变更详情 | 固化变更记录、通知状态、影响范围 | 变更详情 | 有始有终 |
| 顶部通知入口 | warning / breaking 变更 | 通知抽屉、标记已读、重试 Webhook | 通知已读、发送成功、失败可重试或 not_configured | 变更详情 | 有始有终 |
| 我的团队 / 项目详情 / MCP | 团队管理员创建 Token | MCP 配置、工具范围、当前激活项目 | IDE/Agent 可按 Token 访问当前激活项目的本地 MCP 服务；Git Hook/CI 使用 REST 凭据 | MCP 接入说明 | 有始有终 |
| 权限兜底 | 任意页面或操作 | 平台角色 + 团队角色校验 | 允许、隐藏、置灰或拒绝访问 | 当前页面错误提示 | 有始有终 |

树结构结论：

- 启动链路有明确起点：打开 Electron。
- 团队业务有明确前置条件：必须存在 currentTeamId 和团队角色。
- 团队内所有资产节点都能回到列表、详情或来源页面。
- 导入、变更检测、变更记录、通知、MCP 消费形成完整闭环。
- 平台管理不承载 MCP 配置；MCP 配置跟随当前项目上下文。
- 失败路径均停留在当前节点并给出错误提示，不跳到无上下文页面。

## 3. 全局布局

### 3.1 顶部导航

用途：承载产品级导航、当前团队上下文、本地 MCP 服务状态和用户入口。

组件：

- Synkord Logo / 产品名
- 工作台入口
- 项目空间入口
- 全局搜索入口，可选
- 本地 MCP 服务状态
- 通知入口
- 当前用户头像 / 用户菜单
- Electron 窗口控制按钮

字段：

- 当前团队名称
- 当前登录用户
- 本地 MCP 服务状态：运行中 / 已停止 / 未配置 / 异常
- 当前项目 MCP 配置状态：已配置 / 未配置 / Token 已停用

交互：

- 点击 Logo 回到工作台。
- 点击工作台进入全局首页。
- 点击项目空间进入当前团队空间。
- 点击本地 MCP 服务状态查看当前设备的服务可用性和接入端点。
- 点击通知查看 breaking change 通知。
- 点击用户头像展开个人信息、退出登录。
- Electron 环境下支持最小化、最大化、关闭窗口。

### 3.2 侧边菜单

用途：承载团队上下文和团队内核心业务模块导航。

菜单结构：

```text
左侧菜单
├─ 我的团队
│  ├─ 默认团队
```

菜单关系：

| 团队首页 | 团队首页 | 当前团队 |

| 接口管理 | 接口列表 | 当前团队 |

交互：

- 点击团队切换当前团队上下文。
- 点击菜单切换内容页面。
- 当前菜单高亮。
- 无权限菜单隐藏或置灰。

### 4.1.1 创建团队引导

出现条件：

```text

├─ 标题与说明
├─ 创建团队表单
│  ├─ 团队名称
│  └─ 团队描述
```

| 字段 | 类型 | 必填 | 校验 |
| --- | --- | --- | --- |

用途：作为项目详情中的 MCP 能力页，管理当前项目如何通过本机唯一 MCP 服务暴露给 IDE/Agent。用户打开哪个项目，IDE 连接 Synkord MCP 时就读取哪个项目的规范。

| 项目 MCP 管理 | 查看接入地址 | IDE 接入说明 |

MCP 可用性设计：

- 本地 MCP 服务开关：位于 Electron 本地管理能力中，控制当前设备上的唯一 MCP 服务是否对 IDE/Agent 提供能力。
- MCP 激活上下文：由 Electron 当前打开的团队和项目决定，同一时间只允许一个团队和一个项目对外提供 MCP 规范上下文。
- 单个 MCP 配置开关：每个 Token / 接入配置可单独启用或停用。
- 当前项目未激活或本地 MCP 服务未运行时，IDE/Agent 不能消费规范上下文。
- 当单个配置关闭时，仅该 Token 或该 IDE 配置不可访问。
- 打开项目详情的 MCP Tab 时，Electron 自动把该项目设为当前 MCP 激活项目；如果服务未启动，页面显示启动入口。
- 从项目 A 切换到项目 B 时，顶部轻提示“当前 MCP 项目已切换为 B，IDE 后续请求将使用该项目规范”。

本地 MCP 连接策略：

- 当前设备只运行一个 Synkord 本地 MCP 服务入口。
- 本地 MCP 服务入口同一时间只绑定 Electron 当前打开的一个团队和一个项目。
- 切换团队或项目时，Electron 更新本地 MCP 服务的激活上下文；IDE/Agent 后续 MCP 请求使用新的上下文。
- IDE 配置中的本地 MCP 地址保持稳定；项目切换不要求用户修改 `.mcp.json`。
- 本地 MCP 服务根据当前激活团队、当前激活项目和 Token 请求后端校验用户、团队、项目和工具权限。
- 每个项目可以创建多个 MCP Token，用于区分 Codex、Cursor、VSCode、PyCharm、JetBrains 等不同 IDE/Agent。
- 每个 Token 可配置名称、用途、权限范围、可用工具、过期时间和启用状态。

MCP 配置字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| name | 文本 | 配置名称，例如 Cursor 开发环境 |
| token | 密钥 | MCP 访问 Token，仅创建时完整展示 |
| teamId | 关联字段 | 所属团队 |
| projectId | 关联字段 | 所属项目，创建后不通过 Token 切换 |
| toolScope | 多选 | 允许调用的 MCP 工具 |
| status | 状态 | 启用 / 停用 |
| expiresAt | 时间 | 过期时间，可为空 |
| lastUsedAt | 时间 | 最近调用时间 |
| createdBy | 用户 | 创建人 |
| createdAt | 时间 | 创建时间 |

MCP 访问状态规则：

| 本地 MCP 服务 | 当前项目上下文 | Token 状态 | 访问结果 |
| --- | --- | --- | --- |
| 已停止 | 任意 | 任意 | 拒绝访问 |
| 运行中 | 未激活项目 | 任意 | 拒绝访问 |
| 运行中 | 已激活项目 | 停用 | 拒绝访问 |
| 运行中 | 已激活项目 | 启用 | 允许访问授权工具 |

MCP 操作交互：

- 未启动本地 MCP 服务时，显示“启动本地 MCP 服务”主按钮。
- 本地 MCP 服务已运行但当前项目未激活时，显示“设为当前 MCP 项目”主按钮。
- 当前项目未创建 Token 时，显示“创建 Token”主按钮。
- 点击新建 MCP 配置：打开配置弹窗，填写名称、用途、工具范围、过期时间。
- 创建成功后仅展示一次完整 Token，并提供复制按钮。
- 点击停用配置：弹出确认框，确认后该 Token 不能继续调用 MCP。
- 点击重新生成 Token：弹出高风险确认，确认后旧 Token 立即失效。
- 删除配置：仅删除配置记录和 Token，不删除调用审计。

MCP 权限规则：

- 团队管理员可以为当前项目新建 Token、停用 Token、重新生成 Token。
- 编辑者可以查看 MCP 接入说明和调用审计，不可管理 Token。
- 只读者默认不可查看 Token，可查看公开的 IDE 接入说明。
- MCP 页面展示当前设备的 MCP 接入地址、当前激活团队、当前激活项目和状态为只读信息；本地服务配置由 Electron 管理。
- 当前激活项目是 Electron 运行时上下文，不是 Token 字段；Token 只能绑定创建它的项目。

IDE 配置示例：

```json
{
  "mcpServers": {
    "synkord": {
      "url": "${SYNKORD_MCP_URL}",
      "headers": {
        "Authorization": "Bearer ${SYNKORD_MCP_TOKEN}"
      }
    }
  }
}
```

该配置只连接本地 MCP 服务，不表达团队或项目。团队和项目由 Electron 当前激活上下文决定。

### 5.2 用户管理交互

搜索筛选：

1. 输入关键词或选择筛选条件。
2. 点击查询或回车查询。
3. 前端组装查询参数。
4. 请求用户列表接口。
5. 表格刷新。
6. 分页回到第一页。

邀请 / 新增成员：

1. 点击新增成员。
2. 选择邀请已有用户或创建本地用户。
3. 填写邮箱、角色和必要的账号信息。
4. 校验邮箱是否已在当前团队内存在。
5. 如果用户已存在，创建 TeamMember 关系并发送邀请或直接加入团队。
6. 如果创建本地用户，创建账号后自动加入当前团队。
7. 成功后关闭弹窗并刷新成员列表。

批量删除：

1. 勾选表格行。
2. 显示已选择数量。
3. 点击批量删除。
4. 弹出确认框。
5. 确认后调用删除接口。
6. 删除成功后刷新列表。

单行编辑：

1. 点击操作列编辑。
2. 打开编辑弹窗。
3. 表单自动填充当前用户数据。
4. 修改角色、状态或备注。
5. 点击保存。
6. 校验通过后提交。
7. 成功后关闭弹窗并刷新表格。

单行禁用 / 启用：

1. 点击操作列禁用或启用。
2. 弹出确认框。
3. 用户确认。
4. 调用状态更新接口。
5. 表格状态更新。
6. 展示成功提示。

分页切换：

1. 点击页码、上一页、下一页或修改每页条数。
2. 前端更新 page / pageSize。
3. 请求列表接口。
4. 表格刷新。

弹窗表单校验：

- 用户名在创建本地用户时必填，邀请已有用户时不需要。
- 邮箱必填且格式正确。
- 密码在创建本地用户时必填，邀请已有用户时不需要。
- 角色必选。
- 用户名不能重复。
- 邮箱不能在当前团队内重复。
- 备注长度不能超过限制。

限制规则：

- 当前用户不能禁用自己。
- 当前用户不能删除自己。
- 最后一个团队管理员不能被降级、禁用或删除。
- 只读用户无新增、编辑、禁用、删除和批量删除权限。

| 变更检测 | 新旧 OpenAPI / JSON Schema | info / warning / breaking 结果 | 是 |
| 变更记录 | 检测结果 | 变更详情、通知状态、影响项目 | 是 |
| 通知闭环 | warning / breaking 记录 | 顶部通知、Webhook 状态、可重试失败通知 | 是 |
| MCP 消费 | 当前项目 Token、工具范围 | IDE/Agent 可查询规范和变更 | 是 |
| 权限兜底 | 平台角色、团队角色 | 菜单、按钮、接口权限控制 | 是 |
